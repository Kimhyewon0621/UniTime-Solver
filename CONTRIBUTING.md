# Contributing to UniTime-Solver

Thank you for considering contributing to our project!!
We welcome contributions from everyone. 
This document outlines the process for reporting bugs and submitting code.

## Bug Report & Feature Requests üêõ
If you find a bug or have an idea for a new feature, please use the **Issues** tab.

- Bug Report: Please describe the error and how to reproduce it.
- Feature Request: Please explain the new feature and why it is needed.

## Pull Request Process
To contribute code, please follow these steps:

1. **Fork** the repository to your own GitHub account.
2. **Create a Branch** for your changes.
    *  Naming Convention: Please follow our branch naming rule: `iss[Issue Number]-[Initials]`. üëæ
    *  Example: `git checkout -b feature/add-new-schedule`
3.  **Commit** your changes with a clear message.
    * Example: `feat: Add function to search lecture by time`
4.  **Push** your branch to your forked repository.
5.  **Open a Pull Request (PR)** to the `main` branch of this repository.
6.  **Review & Merge**: The Pull Request must receive approvals from a **all 3 reviewers**.
    * Once all approvals are secured, the PR author is responsible for merging their changes into the `main` branch.
